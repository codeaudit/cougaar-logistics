This document describes how to run the GLMTrans society.

To run the datagrabber and the TPFDD Viewer, please see the datagrabber user's guide.  This is in the doc
directory of the datagrabber module.

NOTE : Commands are given in Unix format but are adapable to NT.

I.  Running a GLMTrans society

   A.  If running with a separate demand society, be sure
       the alpreg.ini is set to the right host machine.  alpreg.ini
       is in $COUGAAR_INSTALL_PATH/configs/common

   B.  Go to the appropriate configs directory

   > cd $COUGAAR_INSTALL_PATH/glmtrans/configs/demo-config/configs

   C.  Execute the GLMTrans Node

   > Node GLMTransNode         

If running the GLMTrans society with a demand society, you'll need to open a separate
window and run a separate demand node.  For example, if running with the small-135 society:

   A.  Go to the appropriate configs directory

   > cd $COUGAAR_INSTALL_PATH/configs/small-135

   C.  Execute the SMALL-135-NODE Node

   > Node SMALL-135-NODE

If you run the SMALL-135-TRANS-NODE, no interesting transportation will occur, because
the SMALL-135-TRANS-NODE society contains a stub agent that acts as a sink for transportation
tasks.  This stub intercepts tasks that the GLMTransNode society would normally receive.

II.  Running a GLMTrans society in testing, standalone mode

You can also run the GLMTrans society without the demand society.  This requires using the InputTaskProvider agent.
This agent uses the GLMStimulatorPlugin to read in tasks from XML files.  There are a variety of these tasks in the testinput
directory.  These test various types of transportation tasks, for instance sea and air tasks, theater-only movement, etc.
