The TPFDD Viewer is a GUI that displays time-phased force deployment data.  Basically, this is
what-when-how-whom information (why is implicit).  That is, the viewer shows what cargo is being transported,
when it departs and when it arrives, how it gets to its destination (on what conveyance), and who owns the
cargo.  The viewer displays this information, enables the user to filter and sort this information, and view
this information from the point of view of the cargo or the conveyances that transport the cargo. The user's guide to the datagrabber and TPFDD Viewer is here .

Runs

Every run that the datagrabber has collected is presented in the Run Menu at the top of the display. Each
run shows the number and time of the run.  The run status (W - warning, OK - no warnings) is presented,
along with an indicator for which run is currently being displayed.  This is consistent with the textual status
information on the right upper side of the main display.

GUI Screens

The viewer presents a number of basic screens : the Log Plan View , the Asset Usage View,
the Run Statistics Tab, the Run Charts Tab, and the Filter Dialog.

Log Plan View

The Log Plan View presents the hierarchy of organizations, with rolled-up itinerary information
presented for each unit or higher-level organization.  The user can drill down into this information until the unit level is reached. There,
the user can choose to see the unit equipment either by cargo or by carrier.  

    By Cargo

The by cargo option shows a list of the types of cargo that this unit owns.  Drilling down into a cargo type, the
list of actual cargo items is presented.
Drilling down to the final level shows each leg of the journey, with the width
of a lozenge indicating the leg's time duration.  Each row also shows from and to information for the cargo
item or a leg of the journey, depending on the level of detail.

    By Carrier

The by carrier option shows a list of the types of carrier (or conveyances) that were used to transport the unit equipment.
Drilling down further, the list of actual carriers of the type is shown. If the type name is colored yellow, it means that the
equipment is self-transportable, that is, it's a truck or other conveyance that transports itself and is not moved on land by
another conveyance. Drilling down to the next level, a list of cargo types
carried by the conveyance is shown.  At this point, the display is like the by cargo display, showing the items carried.

    Right Click Options

        At any row of the Log Plan View, you can right click, and you will be presented with a menu that allows the
user to choose from a number of other displays.  The choices are :

        New TPFDD for XXX

            where XXX is item being selected.  XXX can be as large as a unit or as small as an individual item of cargo.  

        Select Filter

            This brings up a filter dialog, allowing the user to sort through the transportation information

        Carrier Use (only in a carrier context)

            If the item selected is a carrier, it will show a TPFDD-like display that shows the use of the carrier or carrier
    type over the course of the whole move.

        Display Route (only in a cargo or carrier context)

            Displays on a map the route the cargo item or carrier follows.

        Display Details (only in a cargo or carrier item context)

            Displays other details about the carrier or cargo, depending on context.  Included is information about dimensions, ownership, etc.

Colors

The colors in the Log Plan View indicate what mode the cargo went by : air (blue), sea (green), or ground (yellow),
or some combination (purple).

TPFDD Display

The TPFDD display has rows for each item, where each row shows a sequence of symbols that represent the
stages of the journey the cargo takes. The width of each lozenge or symbol indicates the amount of time taken
on that leg.  The color coding indicates what mode the cargo item went by : green for sea, yellow for ground, blue
for air.  

Mousing over each lozenge item, a textual description appears on the status line below.  This presents in English
the meaning of the lozenge.  For instance, mousing over the first lozenge item may show something like :

Origin : FT_STEWART departs at 9/12/01 10:52

This indicates the origin of the item being moved is Fort Stewart, Georgia and the date of its departure.

The dialog allows the user to zoom in or zoom out, or to display more or fewer rows in the dialog at any one time.
By dragging the date numbers in blue at the top, the user can look at different time windows.

As the user zooms in or out, more or less textual information is displayed in the lozenges, depending on how much
can fit inside the lozenge.

    Status Line

Below the dialog controls and above the TPFDD lines is a status line that gives four pieces of information about the TPFDD:
1) Total number of items in TPFDD
2) Total tonnage (short tons)
3) Total volume (cubic meters)
4) The time it took to create the TPFDD.

    Export menu

TPFDD data can be exported to a CSV (comma separated values) file that's convenient to manipulate in Excel.  There
are two options : Export CSV file and Export Sea Legs CSV file .  The Sea Legs option filters the TPFDD data so that only
sea legs are sent to the CSV file.  When a menu item is selected, a save dialog appears asking for the name of the file to save to.

Filtering

    TPFDD Filter Dialog

The TPFDD Filter Dialog allows you to select a unit or some set of units and filter by carrier type or instance and/or
cargo type or instance.  The results can then be rolled up by cargo type, or by unit and then by cargo type.

On the left side is the unit hierarchy. Once a unit or units is selected, you can then click Get Manifest Info to fill in
the list boxes on the right.  These are then populated with the types of carriers and the actual carriers used for
the selected units, and the cargo types and cargo instances that belong to the selected units.  At any time the user
can click the Display TPFDD button to view a TPFDD display.  The Aggregate TPFDD button brings up a TPFDD-like
display where each row presents an aggregate row for all instances of the type.  The Unit Aggregate button breaks
this information down first by unit and then by cargo type.

For instance, this dialog allows the user to ask which ship carried the tanks of the 3-69-ARBN battalion.
Or more specifically, which ship carried a specific tank, or which items were on a specific ship.

On the left side, once a unit is selected, clicking the Select Subordinates button will select all subordinates of the unit. The
check boxes on the upper right control which criteria, carrier and/or cargo filtering is to be used.

The Sort By Name check box controls whether the TPFDD generated is sorted by time or by cargo name.

Asset Usage View

The Asset Usage View shows the use of assets over the course of the whole move.  This display is from the point of
view of the carrier.  The carrier assets are broken down by type -- sea, air, land, land (self-propelled), and misc.  Within
each category, they are further broken down by carrier type. For instance, within air, there are 747s, C-17s, etc.  These are
then broken down by where the assets are based or what organization owns them. Drilling down further, there are rows for
each individual asset.  Below this level, there is a row for each mission undertaken by the asset.  And below this level, there
is a row for each leg of the mission.  Here too, one can right click and see details about the carrier, including a TPFDD view
to see what items are carried on the asset over the course of the whole move.

Run Statistics

The run statistics tab allows the user to ask very specific questions about the data.  For instance, the user can ask how many items
were shipped for per unit. Other options include presenting the cargo dimensions for all cargo types, or arrival time precision per unit
(this shows how close to the requested arrival time cargo items were actually scheduled).

Run Charts

This option presents the statistics information in chart form.
